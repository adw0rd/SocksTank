# RPi 5 — питание и потребление

Дата замера: 2026-02-25, обновлено 2026-02-27.

## Проблема

RPi 5 (Cortex-A76 2.4 GHz, 8 GB RAM) потребляет значительно больше энергии, чем RPi 4.
Freenove Tank Board рассчитана на RPi 3/4 (~2-3A max), DC/DC конвертер на плате не справляется с пиковым потреблением RPi 5.

### Симптомы

- `Undervoltage detected!` в journalctl
- `throttled=0x50005` (бит 0 — текущий undervoltage, бит 16 — был undervoltage)
- Зависания и крэши при пиковой нагрузке (компиляция, все 4 ядра)
- EXT5V_V падает до 4.73V (норма: 5.0-5.1V)

## Источники питания (тесты)

| Источник | EXT5V_V | Результат |
|---|---|---|
| Freenove Tank Board (2x18650 через DC/DC) | ~4.8-4.9V | Undervoltage при нагрузке, крэши |
| Xiaomi 120W USB-C зарядка | 4.73-4.86V | Undervoltage (нет PD negotiation, ~5V/2-3A) |
| Телефонная 3A USB-C зарядка | ~4.8V | Undervoltage |
| Официальный RPi 5 PSU (27W USB PD) | 5.0-5.1V | ✅ Рекомендуется |

## Решение: XL6019E1 DC-DC Buck-Boost Converter ✅

**XL6019E1** (5A, buck-boost) — рекомендуемый конвертер для RPi 5.

### Схема

```
Батареи (2x18650, 7.4V) → XL6019E1 (5.2V) → RPi 5 GPIO (Pin 2 = 5V, Pin 6 = GND)
```

### Параметры XL6019E1

- Вход: 5-32V (подходит для 2S LiPo 6.4-8.4V)
- Выход: регулируемый, настроить на **5.2V** (потенциометр)
- Макс. ток: **5A** (хватает для RPi 5 на 4 ядрах)
- Тип: buck-boost (повышающе-понижающий)

### Конденсаторы (рекомендуется)

Для стабилизации пиков тока при загрузке RPi 5 припаяны два электролитических конденсатора:
- **Вход** (сторона батарей): 1000-2200µF 16V
- **Выход** (сторона 5.2V): 2200µF 10V

#### Результаты до и после конденсаторов

| Параметр | До | После |
|---|---|---|
| EXT5V_V (idle) | 5.220V | 5.226V |
| EXT5V_V (4 ядра) | 5.22-5.23V | 5.22-5.23V |
| Пиковый старт (4 ядра без плавного) | ❌ крэш | ✅ стабильно |

> Конденсаторы сглаживают бросок тока при холодном старте и переключении нагрузки.
> Без них прямой запуск на 4 ядрах крашил rpi5 (пиковый ток >5A).

### Сравнение конвертеров

| Конвертер | Тип | Rated | Выход | 1 ядро | 2 ядра | 3 ядра | 4 ядра |
|---|---|---|---|---|---|---|---|
| LM2596 | buck | 3A | 5.1V | ✅ 7.7 FPS | ✅ 11.1 FPS | ❌ крэш | ❌ крэш |
| **XL6019E1** | buck-boost | **5A** | 5.2V | ✅ | ✅ | ✅ | ✅ (плавный старт) |

> LM2596 (3A) не держит 3+ ядер — ток превышает rated. XL6019E1 (5A) — рекомендуется.

### Рекомендации

1. Настроить выход на **5.2V** (проверить мультиметром перед подключением)
2. Подключить через GPIO Pin 2 (5V) и Pin 6 (GND)
3. Припаять конденсаторы (вход + выход)
4. Использовать **плавный старт** при загрузке модели (1→2→3→4 ядра)
5. Проверить: `vcgencmd pmic_read_adc | grep EXT5V` → >5.0V
6. Максимальное допустимое напряжение RPi 5: **5.5V** (рекомендуется 5.0-5.25V)

> ⚠️ **Разряженные 18650** (суммарно <6.5V) вызывают crash-loop при загрузке. Проверять напряжение батарей мультиметром!

## config.txt настройки питания

```
# /boot/firmware/config.txt
usb_max_current_enable=1
psu_max_current=5000
```

Эти параметры разрешают RPi потреблять до 5A через GPIO/USB.

## Workaround (временный)

Если нет XL6019E1, можно ограничить нагрузку:
```bash
# Ограничение на 1-2 ядра (безопасно для LM2596 3A)
taskset -c 0,1 python3 main.py serve ...
```
