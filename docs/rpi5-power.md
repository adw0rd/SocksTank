# RPi 5 — питание и потребление

Дата замера: 2026-02-25.

## Проблема

RPi 5 (Cortex-A76 2.4 GHz, 8 GB RAM) потребляет значительно больше энергии, чем RPi 4.
Freenove Tank Board рассчитана на RPi 3/4 (~2-3A max), DC/DC конвертер на плате не справляется с пиковым потреблением RPi 5.

### Симптомы

- `Undervoltage detected!` в journalctl
- `throttled=0x50005` (бит 0 — текущий undervoltage, бит 16 — был undervoltage)
- Зависания и крэши при пиковой нагрузке (компиляция, все 4 ядра)
- EXT5V_V падает до 4.73V (норма: 5.0-5.1V)

## Источники питания (тесты)

| Источник | EXT5V_V | Результат |
|---|---|---|
| Freenove Tank Board (2x18650 через DC/DC) | ~4.8-4.9V | Undervoltage при нагрузке, крэши |
| Xiaomi 120W USB-C зарядка | 4.73-4.86V | Undervoltage (нет PD negotiation, ~5V/2-3A) |
| Телефонная 3A USB-C зарядка | ~4.8V | Undervoltage |
| Официальный RPi 5 PSU (27W USB PD) | 5.0-5.1V | ✅ Рекомендуется |

## Решение: LM2596 DC-DC Buck Converter

**План:** Установить LM2596 (или LM2596S) между батареями и RPi 5 GPIO.

### Схема

```
Батареи (2x18650, 7.4V) → LM2596 → 5.1V/3A → RPi 5 GPIO (5V + GND)
```

### Параметры LM2596

- Вход: 4.5-40V (подходит для 2S LiPo 6.4-8.4V)
- Выход: регулируемый, настроить на **5.1V**
- Макс. ток: 3A (достаточно для RPi 5 без USB нагрузки)
- КПД: ~80-85%
- Стоимость: ~$1-2

### Рекомендации

1. Настроить выход LM2596 на **5.1V** (не 5.0V — запас на потери в проводах)
2. Подключить через GPIO pins 2 (5V) и 6 (GND)
3. Добавить электролитический конденсатор 100-470µF на выходе для стабилизации
4. Проверить: `vcgencmd pmic_read_adc | grep EXT5V` → должно быть **>5.0V**
5. Проверить: `vcgencmd get_throttled` → должно быть **0x0**

### Альтернативы

| Конвертер | Ток (макс) | Цена | Примечание |
|---|---|---|---|
| LM2596 | 3A | ~$1-2 | Достаточно для RPi 5 без USB |
| XL4015 | 5A | ~$2-3 | С запасом, для USB-периферии |
| MP1584 | 3A | ~$0.5 | Меньше размер, менее стабильный |
| Pololu 5V 5A Step-Down | 5A | ~$15 | Высокое качество, малый размер |

## config.txt настройки питания

```
# /boot/firmware/config.txt
usb_max_current_enable=1
psu_max_current=5000
```

Эти параметры разрешают RPi потреблять до 5A через GPIO/USB.

## Workaround (временный)

Пока нет LM2596, можно ограничить нагрузку:
```bash
# Компиляция на 1 ядре с низким приоритетом
taskset -c 0 nice -n 19 <command>
```
